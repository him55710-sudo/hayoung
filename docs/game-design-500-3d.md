# 500일의 방: 1인칭 3D Escape UX Draft

## Core Experience

이 프로젝트는 일반 웹페이지가 아니라 풀스크린 1인칭 3D 방탈출 게임처럼 시작한다. 사용자는 첫 화면에서 질문을 받고, `네` 버튼은 잠시 도망가다가 6초 뒤 클릭 가능해진다. 이후 플레이어는 하영이의 시점으로 5개의 테마 방을 차례로 지나며 총 10개의 문제를 푼다.

## Entry Interaction

- Copy: `임현수와의 500일을 함께하실 준비가 되셨나요?`
- Desktop: 커서가 가까워지면 `네` 버튼 위치가 바뀐다.
- Mobile: 터치가 닿으면 버튼이 미끄러지듯 이동한다.
- 6초 후 버튼은 제자리로 돌아오고 클릭 가능해진다.

## Game Structure

총 5개 방, 방마다 2문제, 총 10문제.

1. `1-100일`: 풋풋하고 밝은 시작의 방
2. `101-200일`: 조금 성숙하고 밝아진 방
3. `201-300일`: 고난과 화해의 방
4. `301-400일`: 지치고 다사다난하지만 서로 좋은 방
5. `401-500일`: 구름길, 섬광, 시간순 사진, 편지 엔딩

## Puzzle Dependency

현재 초안은 한국 방탈출 카페의 연속형 문제 흐름을 따라 선형 의존 구조로 잡았다.

```text
1 첫 자물쇠
 -> 2 첫 사진 프레임
 -> 3 카페 방향 패널
 -> 4 약속의 별
 -> 5 비 오는 날의 금고
 -> 6 깨진 소리 방향
 -> 7 지친 밤의 선택
 -> 8 열린 창문의 신호
 -> 9 500일 하늘문
 -> 10 편지의 마지막 문장
```

## Puzzle Types

- 숫자 자물쇠: 4자리 날짜/순서 코드
- 방향 자물쇠: 위, 아래, 왼쪽, 오른쪽 순서
- 상징 장치: 별, 달, 빛, 사진 등 반복 상징
- 기억 단서: 실제 사진이나 에피소드 기반 선택
- 장치형 문제: 물체를 올리거나 순서를 맞추면 빛/소리로 반응하는 구조
- 입력 장치: 숫자 자물쇠는 키패드, 방향 자물쇠는 십자 패드, 기억/상징/최종 문제는 선택 칩으로 조작한다.

## HUD

- Top-left: `500일의 방`
- Top-center: `Room 1/5`, `Puzzle 0/10`, `Hints 3`
- Top-right: 힌트, 배경음, 전체화면, 처음부터
- Center: 작은 1인칭 레티클
- Left: 현재 방 제목/기간/감정 설명
- Bottom: 10칸 인벤토리
- Bottom-right: `E 조사하기` 또는 `다음 방`
- Mobile: 상단 상태/아이콘/방 제목을 분리한 압축 HUD, 하단 좌측 이동 패드, 우측 상호작용 버튼

## Interaction Object Direction

중앙 잠금 장치는 단순 모달 호출 버튼처럼 보이면 안 된다. 플레이 화면 안에서 바로 읽히는 물리 오브젝트여야 한다.

- 방향키 패널: `URDL`류 방향 문제와 연결되는 네 방향 버튼
- 다이얼 링: 숫자 자물쇠/순서 자물쇠를 암시하는 반복 링
- 하트 자물쇠: 연애 기념일 테마를 드러내는 핵심 잠금 오브젝트
- 스캔 라인/상태등: 상호작용 가능 상태와 정답 입력 후 반응을 보여주는 얇은 발광 요소
- 조작대 전면 패널: 큰 검은 덩어리처럼 보이지 않게 목재 패널과 트림으로 나눈다.
- 해제 연출: 정답을 맞히면 콘솔 잠금쇠가 위로 들리고, 볼트가 양쪽으로 밀리며, 문 틈의 빛과 스파크가 다음 단서 방향을 강조한다.

## Hint Penalties

힌트는 최대 3번.

1. 현수한테 바나나우유 사주기
2. 현수한테 설빙 사주기
3. 현수랑 방탈출 하러가기

## Audio Direction

- 1번 방: 따뜻한 아침 피아노 패드
- 2번 방: 카페 느낌의 밝은 마림바/패드
- 3번 방: 비 오는 저음 스트링
- 4번 방: 밤 도시와 심장 박동 같은 패드
- 5번 방: 천국 느낌의 반짝이는 코러스

현재는 절차형 Web Audio로 구현했고, 추후 실제 오픈소스/직접 제작 루프 파일을 넣을 수 있다.

## Visual Direction

- 전체 화면이 곧 게임 화면이다.
- 작은 미니게임 프레임은 금지한다.
- 1인칭 시점으로 방 안의 자물쇠, 장치, 사진, 빛을 직접 보는 느낌을 만든다.
- 데스크톱에서는 하영이의 손, 손전등, 하트 열쇠가 화면 하단에 작게 보여 1인칭 캐릭터 체감을 준다.
- 모바일에서는 조작 버튼과 인벤토리를 우선해 손전등 빔을 숨기고 3D 리그를 낮게 배치한다.
- Roblox처럼 읽기 쉬운 비율은 유지하되, 조명/재질/밀도는 Steam 인디게임에 가깝게 올린다.
- 최종 방은 시간순 사진 프레임 복도, 구름 발판, 중앙 하늘문, 강한 섬광, 편지 오버레이가 핵심이다.
- 방 전환 시 플레이어를 새 방 중앙 입구로 재배치해 벽에 붙은 상태로 시작하지 않게 한다.
